# Phase Wraps

One Markdown file per completed phase, written at the end of `/phase-complete`.

## Purpose

A wrap summary captures the phase's reasoning and outcomes in a form a future contributor (or future-you) can skim in 5 minutes. It is the canonical answer to "why did we build this the way we did?"

## Format

Copy [TEMPLATE.md](TEMPLATE.md) to `phase-N-wrap.md` and fill every section. Do not invent content — if a section doesn't apply, write `n/a`.

## Lifecycle

- Written during `cleanup` stage by `/phase-complete`.
- Never edited after the phase advances, except to fix factual errors.
- Not intended for marketing or external communication; be blunt and honest.
